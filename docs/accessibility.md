# Accessibility Guidelines

BMD components render accessible semantic markup by default.

- Use native buttons for actions.
- Add `aria-expanded` and `aria-controls` for disclosure controls.
- Use `role="tablist"`, `role="tab"`, and `role="tabpanel"` for tabs.
- Keep visible labels as accessible names.
- Use semantic regions: `aside`, `nav`, and `section`.
- Do not require hover-only interaction.
- Preserve keyboard behavior through native buttons and ARIA states.
- Never hide meaningful content from screen readers unless duplicate text exists.

Raw HTML remains author responsibility.
